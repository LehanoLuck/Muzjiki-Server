# Текущий статус реализации базового игрового процесса

Дата среза: 2026-05-18.

## Что уже реализовано

1. Игроки могут начать матч через очередь quick match (`QueueQuickMatch`), и при наличии двух подключенных игроков создаётся `GameSession`.
2. При создании матча инициализируется состояние игроков (`GameState.CreateInitial`) с колодами, рукой, сбросом и play-zone.
3. Реализованы базовые шаги хода по событиям: `DrawCard`, `PlayCard`, `EndTurn`.
4. Реализовано переключение текущего игрока и счётчик ходов.
5. Реализован базовый урон при резолве: суммарная `Power` карт в play-zone атакующего вычитается из `Health` оппонентов.
6. Добавлена синхронизация публичного состояния матча каждому игроку через WebSocket (`GameStart`, `GameStateSync`).
7. В доменных моделях (`GameLoop/Models`) есть заготовки расширенной карточной системы: типы карт, эффекты, валидация карты/эффектов, колода/сброс/play-zone/рука.

## Что пока не реализовано (по вашим пунктам)

1. Полноценный «старт матча» с бизнес-правилами и подтверждением готовности игроков.
2. Выдача стартового набора карт по формализованному правилу (сейчас только один добор верхней карты в `RunStepDraw`).
3. Карточные свойства по ТЗ: стоимость + абилки (добор, сброс, урон, защита, +энергия) в боевой логике.
4. Ход с выбором «либо сыграть карту, либо добрать 1..3 карты по типу карты».
5. Энергия как взаимный общий ресурс между игроками (переток при тратах/получении).
6. Условие завершения игры при `Health <= 0` (сейчас нет окончания матча и winner-state).
7. Формулы по характеристикам:
   - `1 стойкость = +3 к max health`;
   - `скорость = +1 энергия`;
   - мульти-усиление эффектов карт от силы/стойкости.
8. Строгое ограничение play-zone: только 1 карта у игрока одновременно в runtime-состоянии `GameState` (в доменной модели `PlayZone` ограничение уже есть).
9. Полная боевая логика для 4 типов карт: Атака, Защита, Захват, Манёвр.
10. Штраф при замешивании колоды: `-3 HP` с возможной смертью.
11. Правило отставания по энергии на 10+: `10 + сила противника` урона и уменьшение разницы энергии на 10.

## Рекомендуемый следующий этап

1. Вынести боевую логику в отдельный `CombatEngine` (правила, эффекты, win/lose).
2. Перевести `GameLoopService` на операции engine-командами (draw/play/endturn).
3. Ввести `SharedEnergyState` (или аналог), чтобы энергия была не атрибутом одного игрока.
4. Добавить state-machine фаз хода и валидацию разрешённых действий.
5. Закрыть правилами все 11 пунктов и покрыть unit-тестами.
